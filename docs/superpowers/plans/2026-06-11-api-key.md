# Optional LLM API Key Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional `llm_api_key` config field that sends an `Authorization: Bearer <key>` header on every LLM HTTP request.

**Architecture:** Extend `Config` with `llm_api_key: Option<String>`, thread it through `ChatRequest`, and conditionally set the header in `llm.rs` before the `ureq` POST.

**Tech Stack:** Rust, `ureq` v2, `toml`, `serde`

---

## Files changed

- `src/config.rs` — new config field + deserialization + default + tests
- `src/app/llm.rs` — new `ChatRequest` field + header injection + tests
- `src/app/actions.rs` — wire `api_key` into all three `ChatRequest` construction sites
